1.创建maven project的war工程时遇到pom.xml第一行报错Cannot detect Web Project version. Please specify version of Web Project…如下

2.创建Maven项目后pom.xml报错“web.xml is missing and is set to true”

在src下面创建文件夹WEB-INF然后再创建一个名为web.xml的.xml文件即可。

3.在Myeclipse中启动tomcat本地服务器报错： Unsupported major.minor version 52.0

参考stackoverflow:https://stackoverflow.com/questions/22489398/unsupported-major-minor-version-52-0

原因：java的版本不搭配，java的编译环境使用的是JDK1.8，但是runtime运行环境使用的是更低的版本，比如JDK1.7，可在Myeclipse或者eclipse的windows中找到java下面的Complier和Installed JREs中查看，换成相同版本的编译器即可。

7.在PL/SQL中查询数据库，发现日期中的中文乱码，但是其他字段的中文并没有乱码。

解决办法：配置Administrator的用户环境变量NLS_TIMESTAMP_FORMAT YYYY-MM-DD HH24:MI:SS:FF6

点击此电脑右键属性，点击高级系统设置，在弹出的窗口中选高级，下面有个环境变量，点击进去，在Administrator的用户变量下点新建，在变量名弹出的窗口中写入，变量名：NLS_TIMESTAMP_FORMAT ，变量值是：YYYY-MM-DD HH24:MI:SS:FF6。最后关闭PL/SQL客户端再重启即可。

（小编在帖子里面看到说修改注册表中的NLS_LANG ,这个对日期中的中文乱码并没有什么作用（小编亲测），但是它可以解决开启PL/SQL端时提示的数据库字符集和客户端字符集不一致）

8.解决开启PL/SQL端时提示的数据库字符集和客户端字符集不一致

解决办法：按win+R,在弹出的小窗口中输入regedit，然后按回车键，就会跳出来注册表编辑器。按照如下路径计算机\KEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\ORACLE\KEY_XE就可以找到oracle的NLS_LANG。点击NLS_LANG右键，选修改，将弹出的窗口中的数值数据改为SIMPLIFIED CHINESE_CHINA.AL32UTF8，它原来的数据是SIMPLIFIED CHINESE_CHINA.ZHS16GBK点击确定即可。
